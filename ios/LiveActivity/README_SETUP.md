# Live Activity ‚Äî Xcode setup

All Swift code, the Dart bridge, the Info.plist key, and the notifier
lifecycle calls are already in this repo. **What you need to do once in
Xcode** (cannot be done from the CLI without corrupting `project.pbxproj`):

> Requires Xcode 14+, iOS 16.1+ deployment target, **and an Apple
> Developer account** (Live Activities need a signed widget extension).

---

## 1. Add the Widget Extension target

1. Open `ios/Runner.xcworkspace` in Xcode.
2. **File ‚Üí New ‚Üí Target‚Ä¶**
3. Pick **Widget Extension**, click Next.
4. Fill in:
   - **Product Name**: `MyGymBroLiveActivity`
   - **Bundle Identifier**: `<your Runner bundle id>.MyGymBroLiveActivity`
     (Xcode auto-fills this ‚Äî leave the suffix as `MyGymBroLiveActivity`).
   - ‚úÖ **Include Live Activity** (critical ‚Äî adds ActivityKit boilerplate)
   - ‚ùå Include Configuration Intent (not needed)
   - **Embed in Application**: `Runner`
5. Click **Finish**. If Xcode asks to activate the new scheme, click
   **Activate**.

Xcode will create a folder `ios/MyGymBroLiveActivity/` with autogenerated
files. **Delete its autogenerated `.swift` and `Info.plist`** ‚Äî we're
going to use ours instead.

---

## 2. Replace the autogenerated files with the ones already in this repo

In Finder, the files we want are at `ios/LiveActivity/`:

- `MyGymBroActivityAttributes.swift`
- `MyGymBroLiveActivityWidget.swift`
- `Info.plist`

For each file:

1. **Drag it into the Xcode project navigator**, under the
   `MyGymBroLiveActivity` group.
2. In the **Add Files** dialog:
   - ‚úÖ Copy items if needed: **off** (they're already in the repo)
   - ‚úÖ Added folders: **Create groups**
   - **Add to targets** ‚Äî see below per-file:

| File | Runner | MyGymBroLiveActivity |
|---|---|---|
| `MyGymBroActivityAttributes.swift` | ‚úÖ check | ‚úÖ check |
| `MyGymBroLiveActivityWidget.swift` | ‚ùå uncheck | ‚úÖ check |
| `Info.plist` | ‚ùå uncheck | ‚úÖ check |

> The attributes file is intentionally checked into BOTH targets because
> the main app needs to construct it (via `LiveActivityBridge`) and the
> widget needs to read it.

For `Info.plist`, set the widget extension's **Info.plist file** build
setting to point at `ios/LiveActivity/Info.plist`:

- Select the `MyGymBroLiveActivity` target ‚Üí **Build Settings** tab.
- Find **Packaging ‚Üí Info.plist File** and set it to
  `LiveActivity/Info.plist`.

---

## 3. Add `LiveActivityBridge.swift` to the Runner target

This file is already at `ios/Runner/LiveActivityBridge.swift`. If Xcode
hasn't picked it up automatically:

1. Drag `LiveActivityBridge.swift` into the Xcode project navigator under
   the `Runner` group.
2. Add to target: ‚úÖ **Runner** only (not the widget extension).

---

## 4. Set the widget extension's deployment target

1. Select the `MyGymBroLiveActivity` target ‚Üí **General** tab.
2. Set **Minimum Deployments ‚Üí iOS** to **16.1**.

The main `Runner` app can keep its existing deployment target ‚Äî the bridge
itself is annotated `@available(iOS 16.1, *)` and the Dart wrapper returns
`false` from `isSupported()` on older versions.

---

## 5. Add the Live Activities capability

1. Select the `Runner` target ‚Üí **Signing & Capabilities** tab.
2. Click **+ Capability**.
3. There's no explicit "Live Activities" capability ‚Äî instead, Live
   Activities are enabled by the `NSSupportsLiveActivities` Info.plist
   key, which is **already set** to `true` in `ios/Runner/Info.plist`.

If you plan to push remote updates to the activity (not required for v1
‚Äî our setup uses local updates only), add **Push Notifications** here
and configure the APNs key in your Apple Developer account.

---

## 6. Build + verify

1. Pick a real iPhone (iOS 16.1+) ‚Äî **Live Activities do not render in
   the simulator** as of Xcode 16.
2. Select the `Runner` scheme (not the widget scheme).
3. Run on device.
4. Start a workout. You should see:
   - Lock screen: exercise name + set progress + session timer.
   - Pull-down notifications shelf: same card.
   - Dynamic Island (iPhone 14 Pro+): compact pill on the right, expanded
     view on long-press.
5. Complete a set. The card flips to "REST" mode with a self-updating
   countdown.
6. Finish or discard the workout ‚Üí card disappears.

---

## Troubleshooting

**"Live Activity didn't appear"** ‚Äî most common cause: the user has them
disabled in **Settings ‚Üí Face ID & Passcode ‚Üí Allow Access When Locked
‚Üí Live Activities** OR **Settings ‚Üí MyGymBro ‚Üí Live Activities** (per-app
toggle). Our bridge returns `null` from `start()` in that case; the rest
of the workout still works.

**"Crashed on iOS 16.0 or older"** ‚Äî shouldn't happen, the bridge is
`@available(iOS 16.1, *)` and Dart `isSupported()` short-circuits. If
you see this anyway, double-check that nothing inside the widget target
references symbols above iOS 16.1 without a guard.

**"Widget shows the autogenerated 'Hello, World!' UI"** ‚Äî you forgot to
remove the autogenerated `MyGymBroLiveActivityLiveActivity.swift` (note
the doubled "LiveActivity"). Delete it; keep only our two `.swift` files
in the widget target.

**Activity stays on the lock screen after the workout ended** ‚Äî usually
means `LiveActivityService.end()` didn't fire (e.g., the app was force-
killed). Tap the activity to open the app ‚Äî `startSession()` calls
`endInternal()` on the previous activity if one is stale.
